  8. /**
    9. 

  9. * A ResModel provides an resource centric view on an underlying RDF model.
    10. 

  10. * ResModels show information not as statements but as resources with
    11. 

  11. * properties, similar to Jena models. ResModels may create Resources [URI
    12. 

  12. * nodes and bnodes]. Creating a Resource does not make the Resource visible to
    13. 

  13. * the model; Resources are only "in" Models if Statements about them are added
    14. 

  14. * to the Model. Similarly the only way to "remove" a Resource from a Model is
    15. 

  15. * to remove all the Statements that mention it.

  16. * 
    17. 

  17. * When a Resource or Literal is created by a Model, the Model is free to re-use an existing
    18.

  467.     /** 
    468. 

  468.     * Returns a new Model that is the set-union of the model with another model.
    469. 

  469.     * Duplicate statements are removed. If you want to allow duplicates, use addModel() which is much faster.
    470. 

  470.     *
    471. 

  471.     * The result of taking the set-union of two or more RDF graphs (i.e. sets of triples)
    472. 

  472.     * is another graph, which we will call the merge of the graphs.
    473. 

  473.     * Each of the original graphs is a subgraph of the merged graph. Notice that when forming
    474. 

  474.     * a merged graph, two occurrences of a given uriref or literal as nodes in two different
    475. 

  475.     * graphs become a single node in the union graph (since by definition they are the same
    476. 

  476.     * uriref or literal) but blank nodes are not 'merged' in this way; and arcs are of course
    477. 

  477.     * never merged. In particular, this means that every blank node in a merged graph can be
    478. 

  478.     * identified as coming from one particular graph in the original set of graphs.
    479. 

  479.     * 
    480. 

  480.     * Notice that one does not, in general, obtain the merge of a set of graphs by concatenating
    481. 

  481.     * their corresponding N-triples documents and constructing the graph described by the merged
    482. 

  482.     * document, since if some of the documents use the same node identifiers, the merged document
    483. 

  483.     * will describe a graph in which some of the blank nodes have been 'accidentally' merged.
    484. 

  484.     * To merge Ntriples documents it is necessary to check if the same nodeID is used in two or
    485. 

  485.     * more documents, and to replace it with a distinct nodeID in each of them, before merging the
